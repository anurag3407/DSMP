## üí° Key Architecture Decisions

### **Why Hybrid Storage?**
1. **Gas Optimization**: Only IPFS hashes on-chain (~$0.50 per post vs $50+)
2. **Performance**: Fast reads from MongoDB cache
3. **Flexibility**: Can add features without blockchain upgrades
4. **User Experience**: No gas fees for likes/comments

### **Why Cookie-based Auth?**
- More secure than localStorage (httpOnly, SameSite)
- Automatic CSRF protection
- Better mobile support
- Seamless session management

### **Why Pinata IPFS?**
- Reliable gateway URLs
- No need for self-hosted IPFS node
- CDN-backed image delivery
- Simple API

## üîí Security Best Practices

‚úÖ **Never commit `.env` file to Git**
‚úÖ **Use strong JWT_SECRET** (32+ random characters)
‚úÖ **Enable HTTPS** in production
‚úÖ **Whitelist MongoDB Atlas IP** addresses

‚úÖ **Keep private keys** in secure vaults
‚úÖ **Use environment-specific** configurations
‚úÖ **Enable rate limiting** on all endpoints
‚úÖ **Sanitize user inputs** before database operations
‚úÖ **Validate file uploads** (size, type, content)
